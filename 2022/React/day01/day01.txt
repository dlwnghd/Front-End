프론트엔드 구성

	웹 기획자
	: 스토리 보드 작성 화면을 구성하는 사람 [위치, 버튼 위치 페이지 구성 및 흐름도 ]
	: 전반적으로 고객의 요구사항을 분석 해 웹 서비스를 계획하는 사람

	웹 디자이너
	: 완성된 스토리 보드를 기반으로 명확하게 로고, 이미지, 색상, 컨셉 등 디자인을 구성하는 사람

	웹 퍼블리셔
	: 완성된 스토리 보드와 디자인을 가지고 직접 html, css를 이용하여 웹 사이트를 구현하는 사람
	: 간단한 js를 통한 이벤트 구현

	프론트 엔드 개발자
	: js를 통한 비동기 통신 구현 ( promise, async, await )
	: js 심화 과정 컨텍스트의 실행 과정 순서를 파악하여 능숙하게 다룰 수 있어야함
	: 다양한 js 라이브러리(도구)를 신속하게 찾아내어 적재 적소에 이벤트를 구현할 수 있어야 함
	: 번들링 무거워진 프론트 엔드를 필요없는 것들을 제외하면서 보다 가볍게 만드는 작업
	: 백엔드 서버와 통신하여 얻은 데이터를 보여주어야겠죠
	
--------------------------------------------------------- 

	UI
	(USER INTERFACE)
	: 사용자 레이아웃(화면)
	사용자가 제품을 사용할 때 마주하게 되는 면 (폰트, 컬러, 레이아웃)

	UX
	(USER EXPERIENCE)
	: 사용자 경험(행위)
	사용자가 제품을 직간접적으로 사용하면서 얻는 만족감

	=> 검색창을 예시로 든다면, 검색 입력창 버튼은 UI, 버튼을 누르는 행위 자체는 UX
	즉, UI, UX를 고려한다는 것은 사용자가 웹 화면을 보고 이용했을 때 시각적으로 불편함이 없고 제약 사항이 없는지 고려한다는 것

	naver.com을 주소창에 입력했을 때 일어나는 일

	1. naver.com 을 입력
	2. 브라우저가 도메인 (naver.com)을 통해 대응하는 주소
	3. HTTP를 통하여 도메인서버에 맞는 도메인을 찾아내 페이지를 요청
	4. WAS(서버도우미) 데이터베이스에서 데이터를 작업을 처리
	5. 브라우저가 해당 서버에 html을 요청
	6. 브라우저는 웹 페이지 결과물을 파싱 후 출력
